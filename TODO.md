| Inline assembly (`asm fn`)    | A sibling of `extern fn`: where an extern fn's body lives in another object file, an `asm fn`'s body *is* its instructions. `asm fn f(i64 a: rax) -> i64: rax clobbers(rcx, memory) { "..." }` ŌĆö operands pinned with `:`, `clobbers(...)` between signature and block, one string literal per asm line. Lowered to a real internal `alwaysinline` LLVM function wrapping one inline-asm call, so call sites stay ordinary calls and the existing typechecker/symbol-table/call path is untouched: a *declaration* form, not an expression form. Types and registers stay orthogonal (the type drives conversions, the register only pins location). Registers are contextual identifiers, validated against `--target` (`rax` under `--target aarch64-*` is a clean error). `sideeffect` and `~{dirflag},~{fpsr},~{flags}` are compiler-decided ŌĆö omitting the flags triad silently miscompiles any caller holding a live comparison across the block. Went with the core grammar, not the metasystem hook. See doc/compiler/06-codegen.md ┬¦ asm fn Lowering, doc/09 ┬¦ asm fn, handbook ┬¦ `asm fn`. | LOW |
