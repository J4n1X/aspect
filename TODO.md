| `naked fn` | **The freestanding-`_start` language blocker, resolved.** `naked fn name(params) -> R { "asm" ... }` lowers to a real LLVM function carrying the `naked` attribute (no prologue/epilogue) plus `noinline`; its body is one side-effecting inline-asm block (Intel dialect) followed by `unreachable`. With no prologue, arguments stay in their SysV ABI registers (edi, esi, …) and a result leaves through eax — so it is *called like any other function* and an ordinary call site works. This is the piece `asm fn` deliberately could not be: `asm fn` passes operands *into* a function that has a prologue, whereas a naked fn is the whole calling convention. Now `naked fn _start()` can read `argc`/`argv` off the stack and jump to `main`. New keyword `naked`; `naked`+`asm`/`extern` rejected as combining kinds. Verified at **both -O0 and -O2** (the harness requires agreement, which guards against the `unreachable` body being mis-inferred as noreturn). Regression: `tests/programs/naked_fn.ap` (x86-gated), `failures/naked_asm.ap`. See doc/09 § `naked fn`, doc/06 § naked fn Lowering, handbook § `naked fn`. | HIGH |

| Inline assembly (`asm fn`)    | A sibling of `extern fn`: where an extern fn's body lives in another object file, an `asm fn`'s body *is* its instructions. `asm fn f(i64 a: rax) -> i64: rax clobbers(rcx, memory) { "..." }` — operands pinned with `:`, `clobbers(...)` between signature and block, one string literal per asm line. Lowered to a real internal `alwaysinline` LLVM function wrapping one inline-asm call, so call sites stay ordinary calls and the existing typechecker/symbol-table/call path is untouched: a *declaration* form, not an expression form.
